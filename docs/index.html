Why Moldavite

A thinking tool with a material memory

The name is a reminder that useful ideas come from collision: a note meeting another note, a calendar event meeting a plan, your own memory meeting a new question.

Moldavite keeps that material honest. Your source of truth is a folder of real Markdown files. You can sync it with a tool you trust, edit it elsewhere, put it in git, or leave it entirely offline.

Privacy by architecture

Local is the default state

Moldavite has no account system, hosted notes service, analytics, or telemetry. Your notes live in the Forge you choose, and sensitive integrations are narrow, visible, and permission-gated.

Owner-controlled files

Notes are real Markdown with atomic saves and owner-only permissions. Encrypted locking uses AES-GCM with Argon2 key derivation.

Local AI index

The selected model downloads only after consent. Embeddings and queries then run on Apple Silicon Macs; Intel Macs use keyword search. Model files stay outside the Forge.

Narrow plugin powers

Workers have no direct network or app access. The host enforces permissions, exact HTTPS hosts, redirect checks, prompt identity, and Keychain isolation.

An honest network story: Moldavite checks GitHub for signed updates; the community registry contacts GitHub only when you click Browse; semantic models download from Hugging Face only after you opt in; and a plugin can send data only through net.fetch to exact hosts you approved. Publishing a note to WordPress is therefore an explicit action, not a claim that “nothing ever leaves.”
