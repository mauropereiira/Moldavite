Using Moldavite

Everything you need to go from first launch to a connected, private knowledge base.

Getting started

On first launch, Moldavite creates your Forge — the folder where all your notes live — at ~/Documents/Moldavite/. A short onboarding tour lets you pick a custom location instead if you prefer. Every note is a plain Markdown file, so your Forge works with backups, sync tools, and any other editor.

  • Multiple Forges — you can keep several vaults side by side (Work, Personal, …) and switch between them from the dropdown above the sidebar search bar, or via the quick switcher's Switch Forge… action. Each Forge keeps its own pinned notes, recents, and plugins.
  • Daily notes — one click on Today opens the note for the current day, creating it automatically. Daily notes you leave empty are cleaned up automatically, so your Forge never fills with blank files.
  • Weekly notes — click a week number in the calendar to create or open that week's note. Like daily notes, they stay virtual until you write something and are removed if emptied.
  • Standalone notes — regular notes with any title, organized into folders you can create, rename, and drag notes into from the sidebar.

The editor

Moldavite's editor is a rich-text (WYSIWYG) editor built on TipTap, with Markdown shortcuts — type # + space for a heading, - for a list, and so on.

Formatting

  • Headings, bulleted and numbered lists, and task lists with checkboxes.
  • Bold, italic, underline, strikethrough, highlights, and text alignment.
  • Inline code, code blocks, blockquotes, links, and images.
  • A selection toolbar appears when you highlight text, for quick formatting.
  • Type / for the slash menu — insert blocks and run commands (including plugin commands) without leaving the keyboard.

Wiki links & backlinks

Type [[ to link to another note: [[Note Name]], or [[Display text|target-note]] to show different text. The backlinks panel below the editor lists every note that links to the one you're reading, and the graph view shows your whole web of notes at once.

Tags

Add #tags anywhere in a note. Tags get their own section in the sidebar, and you can right-click a tag to rename it across all notes in one step.

Templates

Start notes from templates — built-in or your own, created under Settings → Templates. Templates support variables that are filled in when applied:

VariableExpands to
{{date}}Current date (YYYY-MM-DD)
{{time}}Current time (HH:mm)
{{day_of_week}}Day name (e.g. Tuesday)

Pin up to six templates for quick access from the "Start with a template" screen on empty notes.

Staying organized

  • Quick switcher / command palette — press ⌘ P to jump to any note, run actions (Open Settings, Toggle Timeline, Open Graph, Switch Forge…), rerun recent searches, or trigger plugin commands.
  • Full-text search — ranked content search across every note, with snippets. Locked and trashed notes are excluded.
  • Folders — a drag-and-drop folder tree in the sidebar for hierarchy when you want it.
  • Note colors — assign a color per note to make it stand out in the sidebar.
  • Tabs & pinning — open multiple notes in tabs, drag to reorder, and pin up to five so they're always at hand.
  • Timeline — a chronological feed of notes bucketed by Today / Yesterday / This Week / This Month / Earlier.
  • Shortcut help — press ⌘ ? to see every keyboard shortcut.

Semantic search

Keyword search finds exact words; semantic search finds meaning. Search for "quarterly planning" and surface the note where you only wrote "Q3 roadmap discussion". It's optional, off by default, and runs entirely on your Mac.

Enable it in Settings → AI & Agents → Semantic search. Before anything happens, Moldavite asks for your consent: enabling downloads a small AI embedding model (all-MiniLM-L6-v2, ~97 MB) once from HuggingFace. That download is the only network request the feature ever makes — the model is cached in the app's data directory (never inside your Forge), and afterwards everything runs fully offline. Your notes never leave your Mac, and locked (encrypted) notes are never read or indexed.

Once your notes are indexed (progress is shown live in Settings), two things light up:

  • Semantic mode in search — the sidebar search grows a Keyword / Semantic chip. Flip to Semantic to rank notes by how close their meaning is to your query, with a similarity percentage on each hit.
  • Related notes — a Related section appears next to Linked mentions below the editor, listing the five notes most similar to the one you're reading. It's the serendipity engine: connections you never made explicit with a wiki-link.

The index lives per Forge and keeps itself up to date as you write, rename, or delete notes. If it ever looks stale, the Rebuild index button in Settings re-embeds everything from scratch. Turning the feature off frees the model from memory immediately.

Safety & privacy

Moldavite is local-only: no cloud, no accounts, no telemetry. On top of that, several layers protect your notes from both accidents and prying eyes.

  • Trash with 7-day retention — deleted notes sit in the trash popover with read-only previews. Restore with one click, or let them auto-clean after 7 days.
  • Note locking — lock any note behind a password. Locked notes are encrypted with AES-GCM using Argon2 key derivation, with brute-force rate limiting and a configurable auto-lock timeout.
  • Encrypted backups — export your entire vault as a password-protected encrypted archive from Settings. App settings can also be exported/imported as JSON (notes are never included in the settings file).
  • Import & export — export notes as Markdown, PDF (with page-size and margin options), or plaintext — per note or in bulk.
  • Atomic writes — since v1.5, note saves are written atomically, so a crash or power loss mid-save can't corrupt a note.
  • Automatic signed updates — Moldavite checks GitHub for new releases and updates in place. Every update is cryptographically signed and verified before install, and a "What's New" popup summarizes the changes afterwards.

Syncing your Forge

Your Forge is a plain folder of Markdown files, so it syncs with any file-sync tool — iCloud Drive, Dropbox, Syncthing, or a git repository. External edits show up live thanks to the built-in file watcher. And if a note changes on disk while you have unsaved edits in Moldavite, neither version is lost: the disk version is preserved as a "(conflict)" copy next to the note, your edits are saved, and a toast tells you a conflict copy was created so you can merge at your leisure.

Apple Calendar integration

On macOS, the right panel shows a month calendar and a timeline of your Apple Calendar events next to your notes. The integration is read-only and built on the native EventKit framework: macOS asks for your permission before Moldavite can see any events, the app never modifies your calendar, and nothing is sent anywhere.

Use your Forge as agent memory

Because a Forge is plain Markdown on disk, AI agents that work with files — Claude Code, for example — can read and write your notes directly. There's no export step, no plugin, and no API in between: point the agent at your Forge folder and it sees the same files you do. Edits made by an agent show up live in Moldavite thanks to the file watcher.

To make that hand-off explicit, open Settings → AI & Agents and click Make this Forge agent-ready. Moldavite writes two files to the root of your Forge:

  • AGENTS.md — a generated guide to your vault: the directory layout, daily/weekly/standalone note naming, the YAML frontmatter schema, wiki-link and tag syntax, and ground rules for agents (plain Markdown only, leave .trash/ and .plugins/ alone, never edit encrypted .md.locked files).
  • .gitignore — covers the app-managed folders (.trash/, .plugins/, .index/) and .DS_Store, so a Forge under version control stays clean.

If either file already exists, Moldavite asks before overwriting it, and the settings section always shows whether an AGENTS.md is present in the current Forge.

Your Forge is just files

Everything above operates on plain Markdown files with YAML frontmatter in your Forge. External edits show up live thanks to a file watcher — feel free to point scripts, sync tools, or other editors at the same folder.
