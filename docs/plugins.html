Building Plugins

API v2 adds permissioned Forge reads, allowlisted HTTPS, and plugin-owned Keychain secrets to Moldavite's sandboxed command plugins.

Trust model — read this first

Every plugin runs in its own sandboxed Web Worker with no DOM, Zustand, Tauri IPC, or network globals. Its only app channel is the curated postMessage API. Moldavite enforces every permission on the host side and shows manifest hosts before enabling a plugin. Runtime host requests use a separate app-rendered consent dialog and remain visible with an individual revoke control.

Consent is pinned to SHA-256 of manifest.json + plugin.js. Any edit — including an allowedHosts change — prompts again. Only enable plugins you trust.

Plugin anatomy

<forge>/.plugins/my-plugin/
├── manifest.json
├── plugin.js
└── README.md          # optional
  1. Create the folder in your active Forge.
  2. Add the two required files below.
  3. Open Settings → Plugins, enable it, and review the permission sheet.
  4. Run commands from the palette (⌘ P) or editor slash menu.

Manifest v2

{
  "id": "wordpress-publisher",
  "name": "WordPress Publisher",
  "version": "1.0.0",
  "apiVersion": 2,
  "permissions": ["notes.read", "net.fetch", "secrets", "ui"],
  "allowedHosts": ["public-api.wordpress.com"]
}
Field Required Meaning
id yes Folder-matching lowercase letters, digits, and hyphens; max 64 characters.
name, version yes Display identity and plugin version.
apiVersion yes Use 2. API v1 remains compatible.
permissions no Requested API groups. Commands are always available.
allowedHosts with net.fetch Exact lowercase DNS hosts; no scheme, port, path, or wildcard.

Host matching is exact. Listing api.example.com does not approve its parent, subdomains, or a non-default port. Every subdomain must be listed separately.

plugin.js

export default function register(api) {
  api.commands.add({
    id: 'inspect-first-note',
    label: 'Inspect first note',
    handler: async () => {
      const notes = await api.notes.list();
      const markdown = await api.notes.read(notes[0].path);
      await api.ui.toast(`Read ${markdown.length} characters`);
    },
  });
}

All methods except commands.add are asynchronous host RPC calls.

API v2 surface

Permission Methods Access
always commands.add Add a namespaced palette/slash command.
editor editor.getActiveNote, editor.insertText Active-note path + HTML and cursor insertion.
always ui.prompt One trusted app-rendered form at a time, with the plugin name fixed in the header.
ui ui.toast Info, success, or error notification.
notes.read notes.list, notes.read Curated metadata and unlocked Markdown.
net.fetch net.requestHostAccess, net.fetch Consent and host-performed HTTPS to exact approved hosts.
secrets secrets.get/set/delete This plugin's Keychain namespace.

Notes

notes.list() returns { path, title, kind, folder }. Paths are Forge-relative, kinds are daily, weekly, or standalone, and folders are relative to notes/. Locked placeholders may appear in the list, but notes.read(path) rejects them. Read content is Markdown, unlike the editor API's HTML.

editor.getActiveNote() includes the Forge-relative path alongside title and live HTML. Use that path as a stable key for per-note plugin state.

Host-rendered prompts

const values = await api.ui.prompt({
  title: 'Configure publishing',
  fields: [
    { name: 'site', label: 'Site URL', type: 'url', required: true },
    { name: 'password', label: 'Application Password', type: 'password' },
  ],
  confirmLabel: 'Save',
});
if (!values) return;

ui.prompt needs no manifest permission. Moldavite validates 1–12 text, password, or URL fields, allows only one plugin dialog at a time, and always identifies the requesting plugin in trusted chrome above the plugin-supplied title. Cancel and Escape return null.

Allowlisted network requests

const site = new URL('https://notes.example.com');
if (!(await api.net.requestHostAccess(site.hostname))) return;
const me = await api.net.fetch(`${site.origin}/wp-json/wp/v2/users/me`);

Dynamic host requests use the same validator as allowedHosts: no IP literals, single-label or localhost names, schemes, ports, paths, or wildcards. Approval is saved in the app-side per-Forge plugin grant rather than the manifest, so the consent hash does not change. Denial returns false. The effective fetch allowlist is the union of manifest and user-approved hosts; revoking one under Settings → Plugins → View permissions affects the next request and every redirect immediately. 

const token = await api.secrets.get('wordpress-token');
const result = await api.net.fetch(
  'https://public-api.wordpress.com/rest/v1.1/sites/example.wordpress.com/posts/new',
  {
    method: 'POST',
    headers: { authorization: `Bearer ${token}`, 'content-type': 'application/json' },
    body: JSON.stringify({ title: 'From Moldavite', content: '# Hello' }),
  }
);

Only absolute HTTPS URLs without embedded credentials are accepted. Redirects are manual: Moldavite resolves and re-validates every Location before continuing, permits at most five, and rejects hidden targets. Credentials are stripped across origins. The full chain times out after 30 seconds; streamed responses stop at 10 MiB.

Results contain status, a safe header subset, bodyText, and bodyBase64 for non-text data. Cookies are never exposed.

Keychain secrets

secrets.get(key) returns a string or null; set and delete return nothing. Moldavite stores service Moldavite, account plugin:<plugin-id>:<key>. The host supplies and validates the plugin id, so one plugin cannot retrieve another's values. Secrets are never listed or exported.

Security and consent

The worker proxy provides friendly early permission errors, but the host independently validates every RPC and is the security boundary. Unknown methods are rejected. Disabling a plugin or switching Forges terminates its worker and pending work.

The consent hash covers the raw manifest and code. Editing permissions or any exact network host therefore invalidates the grant and reopens the permission sheet.

User-approved runtime hosts are separate app-side consent: they survive a version/hash re-grant, can be revoked one at a time, and are forgotten on uninstall.

Publish to WordPress

Install Moldavite's first-party Publish to WordPress reference from Settings → Plugins. Its configure command collects a site URL, username, and Application Password in ui.prompt, asks for the derived exact host, verifies /wp-json/wp/v2/users/me, then stores configuration through api.secrets.

The publish command sends the active editor HTML to /wp-json/wp/v2/posts as a draft. A Keychain-backed Forge-path → post-id map makes later publishes use PUT, avoiding duplicates, and success includes the WordPress edit URL. The dependency-free bundled source is also a practical API v2 example.

WordPress.com limitation

Self-hosted WordPress and WordPress.com Jetpack/Atomic sites work with Application Passwords. WordPress.com Simple sites require OAuth and a registered client ID; this plugin intentionally does not embed or fake one and therefore does not support them.

API v1 compatibility

Existing "apiVersion": 1 plugins still load unchanged with commands, editor, and UI APIs, and see api.app.apiVersion === 1. Use v2 only when requesting note reads, networking, or secrets.
