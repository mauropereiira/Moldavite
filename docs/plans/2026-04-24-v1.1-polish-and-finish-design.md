## Architecture decisions

### Sidebar model (B1)

Single left sidebar, collapsible sections. Backlinks live inline below the editor, not in the sidebar. Trash is a footer icon that opens a popover, not a section.

Layout:

```
┌─ Sidebar ──────────────┐
│ [Search]               │
│ ▾ NOTES                │ ← standalone notes
│ ▾ FOLDERS              │ ← tree, drag-and-drop
│ ▾ DAILY                │
│ ▾ TAGS                 │
│                        │
│ [📅 Today]  [＋ New]   │
│ [⚙ Settings]  [🗑 Trash]│
└────────────────────────┘
```

### Folder model (C2)

Nested folders, one folder per note. Folder = directory on disk. No changes to file layout.

### Full-text search

Rust-side via `ignore` + `grep-searcher` + `grep-regex` (the ripgrep stack). Exposed as a new `search_notes_content` Tauri command. Skips `.md.locked`. Frontend debounces 150ms. Results replace sidebar sections while query is non-empty.

### Backlinks

Collapsible panel pinned below the editor. Auto-refresh 500ms after save. Backend commands (`scan_note_links`, `get_backlinks`, `create_note_from_link`) already exist — no backend work.

### Drag and drop

`@dnd-kit/core` for folder tree. Drop targets are folder nodes and sidebar root. Folder-into-own-descendant guarded in frontend + rejected by existing backend path validation.

### Trash popover

Floating popover attached to footer icon. Lists notes deleted in the last 7 days. Click entry → read-only preview modal (editor with `editable: false`, no toolbar). Actions per entry: Restore (uses existing `restore_note` which returns to original path), Delete permanently. Empty trash button at popover bottom.

### Timeline view

Dedicated pane replacing the editor when opened via sidebar footer button. Groups: Today / Yesterday / This week / This month / Earlier. Items sorted by `modified_at` desc. On macOS with calendar permission, EventKit events interleave with notes for that day as greyed pill rows. Uses existing `get_calendar_events` and `request_calendar_permission`.

### PDF export

Right-click sidebar note → Export as PDF. Also editor overflow menu (`⋯`). Uses existing `exportNoteToPdf` in `src/lib/fileSystem.ts` with its already-hardened DOMPurify sanitization. Save path via `dialog.save`.

### Import / Export

Settings → new Data tab. Four buttons calling existing backend commands: `export_notes`, `import_notes`, `export_settings`, `import_settings`. Path selection via `dialog.save` / `dialog.open`. Toast on success/failure.

### Keyboard shortcut help

`Cmd+/` opens modal. Single source of truth: `src/lib/shortcuts.ts` exports typed list `{ keys, description, category }[]`. `useKeyboardShortcuts` hook consumes the same list so the modal and handler cannot drift.

### Custom templates UX

Settings → Templates tab. Built-ins marked read-only. Custom templates editable/deletable. `+ New template` opens mini-editor modal (name, body textarea, documented tokens `{{date}}` / `{{time}}` / `{{title}}`). Save calls existing `save_template`.

### Z-index fix

Define `z-overlay` (Tailwind `z-[9999]`) on popover root. Drop the empty-note template suggestion container's z-index below that. Verify in browser before calling it done.

## Backend changes

### New commands

- `search_notes_content(query: String, max_results: u32) -> Vec<ContentMatch>`
  - Returns `{ filename, title, snippet, line_number, match_count }`
  - Excludes `*.md.locked`
  - Sorted by `match_count` desc
- `delete_template(name: String) -> Result<(), String>` — if not already present

### New Rust deps

- `ignore = "0.4"`
- `grep-searcher = "0.1"`
- `grep-regex = "0.1"`

### New tests

- `search_notes_content` respects `.locked` exclusion
- `search_notes_content` returns correct snippets and counts
- Content search rejects path-traversal in query context (defense in depth)

### Existing commands reused

`move_note`, `move_folder`, `rename_folder`, `scan_note_links`, `get_backlinks`, `create_note_from_link`, `restore_note`, `empty_trash`, `permanently_delete_trash`, `export_notes`, `import_notes`, `export_settings`, `import_settings`, `exportNoteToPdf`, `get_calendar_events`, `request_calendar_permission`, `save_template`.

## Risk register

| Risk | Mitigation |
|---|---|
| Sidebar refactor regresses note click / keyboard nav | Extract components with identical props; manual QA after step 2 before moving on |
| Drag-and-drop corrupts notes via bad move path | Rely on existing backend path validation; frontend guards against self-drop |
| Full-text search slow on large libraries | Rust engine = ripgrep; add `max_results` cap; debounce 150ms |
| Timeline pane blocks editor | Toggle, not replace; keep editor state; tests for open→close→edit |
| PDF export breaks with hardened CSP | Pipeline already shipped in v1.0.3 with DOMPurify; just adding the trigger |
| Drag-and-drop a11y regression | `@dnd-kit/core` has built-in keyboard support; wire it up |
